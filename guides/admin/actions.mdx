# Apigene Actions Documentation

## Overview

The **Actions** feature in Apigene provides a comprehensive view of all user-initiated activities across your connected applications. This dashboard allows you to monitor, search, and analyze every action performed, including user inputs, API responses, status codes, and more. It is an essential tool for auditing, debugging, and understanding how users interact with your APIs and automations.

---

## Key Capabilities

### 1. Action Log Table

- **Comprehensive Listing**: View a paginated table of all actions performed by users or automations.
- **Columns include**:
  - **User ID**: The identifier of the user who performed the action.
  - **Application**: The name of the connected app or API.
  - **User Input**: The input or command provided by the user.
  - **Created At**: Timestamp of when the action was performed.
  - **Duration**: How long the action took to complete.
  - **Status Code**: HTTP status code returned by the API.
  - **Response Length**: Size of the API response.

### 2. Search and Filtering

- **Search by Application Name**: Quickly filter actions by typing the name of the application.
- **Instant Results**: Results update as you type, and you can press Enter to search.
- **No Results Handling**: Friendly messages guide you if no actions match your search.

### 3. Sorting and Pagination

- **Sortable Columns**: Click on any column header to sort actions by that field (ascending or descending).
- **Pagination Controls**: Choose how many actions to display per page (10, 25, or 50) and navigate between pages.

### 4. Action Details Panel

- **Detailed View**: Click any action row to view its full details in the right-hand panel.
- **User Input**: See the exact input that triggered the action.
- **API Response**: Inspect the full JSON response, including status, headers, and body, in a collapsible, easy-to-read format.

### 5. Live Data and Refresh

- **Reload Button**: Instantly refresh the list of actions to see the latest activity.
- **Live Updates**: The dashboard is designed to keep you up to date with the most recent actions.

---

## How to Use the Actions Dashboard

### Accessing the Actions Page

1. Navigate to the **Actions** page from the main menu in the Apigene web portal.

### Searching and Filtering

- Use the search bar at the top to filter actions by application name.
- Press Enter or click the search icon to apply the filter.
- To clear the search, delete the text and press Enter again.

### Sorting and Pagination

- Click any column header (e.g., "Created At", "Status Code") to sort the table by that field.
- Use the pagination controls at the bottom to navigate through pages and adjust the number of actions per page.

### Viewing Action Details

- Click on any action row to select it.
- The right-hand panel will display:
  - The user input that triggered the action.
  - The full API response, including status code, headers, and response body, formatted as JSON.

### Refreshing the Data

- Click the **Reload** button (circular arrow icon) to fetch the latest actions from your connected applications.
